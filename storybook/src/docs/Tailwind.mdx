# Embracing Tailwind CSS in Spawntech Libraries

At Spawntech, we fully embrace Tailwind CSS's utility-first approach in our component libraries, aligning seamlessly with `shadcn/ui` for pragmatic, scalable UI design. Our implementations leverage Radix-UI to ensure that accessibility is at the forefront of every component we build.

## Why Tailwind CSS?

Tailwind CSS allows us to maintain style consistency with minimal effort, ensuring that our components are both beautiful and functionally robust. The utility-first framework makes it incredibly easy to customize components to fit any design specification directly in your markup.

## Integrating with shadcn/ui and Radix-UI

Our library's design philosophy is closely aligned with `shadcn/ui`, providing a cohesive experience for developers by combining our streamlined components with their comprehensive UI toolkit. Radix-UI's accessibility features are integrated into our components, ensuring that they meet the highest standards for accessibility.

- **Tailwind CSS Documentation**: [Visit Tailwind CSS Docs](https://tailwindcss.com/docs)
- **shadcn/ui Documentation**: [Learn More About shadcn/ui](https://github.com/shadcn/ui)
- **Radix-UI for Accessibility**: [Explore Radix-UI](https://www.radix-ui.com/)

## Importing Spawntech Libraries

To use the Spawntech libraries effectively, include them in your project as follows:

```bash
pnpm install @spwntch/tailwind @spwntch/component @spwntch/shell
```

Here’s how to incorporate our libraries into your application:

```bash
import '@spwntch/component';
import '@spwntch/shell';
```

> **NOTE:**
> If you created an app using `pnpm dlx spawn-workspace`, we highly recommend importing **spwntch** packages into your `nextjs` libraries, not directly into your app. Follow the design pattern already established in a spawned-workspace.

## Typical Tailwind Configuration

Below is a typical `tailwind.config.js` that you would use in a Spawntech workspace:

```js
// tailwind.config.js

const { createGlobPatternsForDependencies } = require('@nx/react/tailwind');
const { join } = require('path');

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    join(
      __dirname,
      '{src,pages,components,app}/**/!(.stories|.spec).{ts,tsx,html}'
    ),
    ...createGlobPatternsForDependencies(__dirname),
  ],
  theme: {
    extend: {},
  },
  plugins: [require('@tailwindcss/typography')],
};
```
